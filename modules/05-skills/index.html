1 Skill 是什么？

Skill 是 Hermes 的可复用知识文档——Agent 完成一次复杂任务后，把过程中学到的流程、命令、注意事项保存为 Skill。下次遇到类似任务时自动加载，不用从头摸索。

这是 Hermes 和所有其他 Agent 框架最本质的区别。Claude Code、Codex CLI 都不会从经验中学习——每次都是"第一次做"。Hermes 会积累经验。

🧠 人类类比

像一个新员工入职第一天 vs 工作一年后。Skill 就是 Agent 的工作笔记——踩过的坑、发现的捷径、验证过的流程，都记下来下次直接用。

📂 物理形态

Skill 就是 ~/.hermes/skills/ 下的 SKILL.md 文件。YAML frontmatter + Markdown 正文。Agent 可以创建、修改、删除自己的 Skill。

2 SKILL.md 格式
---
name: my-skill
description: 这个 Skill 做什么、什么场景触发
version: 1.0.0
---

# 技能标题

## 何时使用
触发条件描述

## 工作流程
1. 第一步
2. 第二步

## 常见问题
- 坑 1：描述 + 解决方案
- 坑 2：描述 + 解决方案
💡 好的 Skill 特征：明确的触发条件、步骤编号、具体的命令示例、常见陷阱标注。越具体越好——"用这个命令"比"处理这个问题"有效 100 倍。
3 技能的生命周期
flowchart TD A["Agent 完成复杂任务"] --> B["保存为 Skill
hermes skills install"] B --> C["Curator 后台维护
追踪使用频率"] C --> D{"活跃使用？"} D -->|"是"| E["✅ 保持活跃"] D -->|"否(30天)"| F["📦 标记为 stale"] F --> G{"又过30天？"} G -->|"是"| H["🗄️ 自动归档
可恢复"] G -->|"被再次使用"| E

Curator 是 Hermes 内置的技能生命周期管理器。它自动追踪每个 Skill 的使用频率，闲置太久的标记为 stale，最终归档（不删除，可恢复）。

hermes curator status  # 查看状态
hermes curator run    # 手动触发维护
hermes curator pin NAME  # 锁定重要 Skill（不吃自动清理）
hermes curator restore NAME # 恢复已归档的 Skill
4 Skill vs Project Context
维度SkillProject Context (.hermes.md)
用途可复用的工作流程、领域知识项目级约定、构建规则
加载时机手动 /skill name 或自动匹配进入项目目录时自动加载
范围跨项目通用当前项目独有
生命周期Curator 自动管理手动编辑维护
典型内容"如何部署这个框架" "API 参考""用 pytest 测试" "不要用 print"
🤔 什么时候用哪个？如果某个知识和项目无关、可能在其他地方复用，写成 Skill。如果是单个项目的特定要求，写到 .hermes.md。
5 技能管理命令
hermes skills list       # 列出已安装
hermes skills search QUERY  # 搜索 Hub
hermes skills install ID  # 从 Hub 安装（支持 URL）
hermes skills uninstall N  # 卸载
hermes skills check      # 检查更新
hermes skills update      # 更新过期 Skill
hermes skills config      # 按平台启用/禁用

# 会话内操作
/skill name        # 加载 Skill 到当前会话
/reload-skills     # 重新扫描 skills/ 目录
6 自学习闭环
flowchart TD A["👤 用户提出任务"] --> B["🤖 Agent 执行任务"] B --> C{"任务成功？"} C -->|"是"| D["💾 Agent 保存为 Skill"] C -->|"否"| E["🔧 用户纠正"] E --> F["📝 Agent 修补 Skill"] D --> G["📚 Skill 库增长"] F --> G G --> H["🎯 下次类似任务
自动加载对应 Skill"] H --> B

这个闭环是 Hermes 的核心创新：用户不需要手动教 Agent——Agent 从每次交互中自动积累经验。用 Hermes 越久，它就越了解你的工作方式和偏好。

📌 要点
  • Skill 是 Hermes 的自学习机制——其他 Agent 没有
  • Curator 自动管理 Skill 生命周期，闲置的归档不删除
  • Skill 跨项目复用，Project Context 项目独有
  • 每次成功完成任务后考虑保存为 Skill
