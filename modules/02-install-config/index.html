1 安装 Hermes
一条命令，60 秒搞定

🐧 Linux / macOS / WSL2

# 一行安装
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash

🪟 Windows (PowerShell)

# PowerShell 中运行
iex (irm https://hermes-agent.nousresearch.com/install.ps1)

📦 Hermes Desktop（Windows / macOS）

如果你更喜欢图形界面，可以从 官网 下载 Hermes Desktop 安装包。它自带 CLI + 桌面 GUI。

安装脚本做了什么？
① 下载最新版本 → ② 解压到 ~/.hermes/hermes-agent/ → ③ 创建 ~/.local/bin/hermes 软链接 → ④ 把 ~/.local/bin 加入 PATH
不会动你的系统 Python，不会污染全局环境。

📁 安装后的目录

~/.hermes/ 下的完整结构在模块 1 已详述。安装完成时只有 hermes-agent/ 源码目录——其余目录在首次运行后自动创建。

🔧 更新

hermes update 一键更新到最新版。也可以重新运行安装脚本——它会覆盖旧版本，保留你的配置和会话数据。

🗑️ 卸载

hermes uninstall 干净卸载。如果需要完全清除所有数据，卸载后手动删除 ~/.hermes/。

📌 本节要点
  • 一条命令安装，不污染系统 Python
  • ~/.hermes/ 是数据根目录，~/.local/bin/hermes 是入口
  • hermes update 升级，hermes uninstall 卸载
2 首次运行：hermes setup
装完之后还不能直接用——需要选模型、配密钥。Setup 向导帮你一步步搞定。
# 交互式配置向导
hermes setup

# 跳到指定配置段
hermes setup model # 只配模型
hermes setup terminal # 只配终端后端
hermes setup gateway # 只配平台接入

Setup 向导会问你什么？

步骤问题你做什么
1 选 Provider（模型提供商） 从列表中选择——OpenRouter / Anthropic / OpenAI / DeepSeek / …
2 输入 API Key 粘贴对应 Provider 的 API Key，自动写入 .env
3 选模型 从 Provider 的模型列表中选择默认模型
4 终端后端 local（本机）/ docker / ssh / modal——默认 local
5 平台接入（可选） 配置飞书 / Telegram / Discord 等，跳过就是纯终端模式
🚀 最快上手路径：如果你有 OpenRouter 账号，运行 hermes setup --portal——一套 OAuth 登录搞定模型 + 搜索 + 生图 + 浏览器，零配置。
📌 本节要点
  • hermes setup 引导式配置，不需要手写 YAML
  • 至少要配置 Model（API Key + 模型名），否则跑不起来
  • hermes setup --portal 是一条龙的最快路径
3 config.yaml 深度解析
一行 YAML 一个行为——这里是你控制 Hermes 的总开关

Setup 向导会自动生成 ~/.hermes/config.yaml。你可以用以下命令管理它：

hermes config # 查看当前配置
hermes config edit # 用 $EDITOR 打开编辑
hermes config set section.key value # 命令行改单项
hermes config path # 打印配置文件路径

🏗️ 完整配置结构

model: ← 模型配置（必填）
default: "deepseek/deepseek-chat" ← 默认模型
provider: "custom" ← 使用的 Provider
base_url: "..." ← 自定义端点（仅 custom/openrouter）
api_key: "..." ← 直接写在 config 中（不推荐，放 .env）
context_length: 1000000 ← 模型上下文长度（用于压缩计算）

agent: ← Agent 行为参数
max_turns: 90 ← 单次请求最大 LLM 调用次数
tool_use_enforcement: true ← 强制 LLM 使用工具（不让它只动嘴）

terminal: ← 终端执行后端
backend: "local" ← local / docker / ssh / modal
cwd: "~/" ← 默认工作目录
timeout: 180 ← 命令超时（秒）

compression: ← 上下文压缩策略
enabled: true
threshold: 0.50 ← 使用率超 50% 触发压缩
target_ratio: 0.20 ← 压缩到 20%

memory: ← 持久记忆
memory_enabled: true
user_profile_enabled: true
provider: "builtin" ← builtin / honcho / mem0

delegation: ← 子代理委托
model: "..." ← 子代理用的模型（不设则继承主模型）
max_iterations: 50

approvals: ← 命令审批
mode: "manual" ← manual（始终确认）/ smart（LLM 判断）/ off（跳过）

display: ← 显示偏好
skin: "default"
show_reasoning: false
show_cost: false

⚙️ model

必填段。定义默认模型、Provider、自定义端点。Hermes 运行时从这里读取"用哪个 LLM"。

provider 可以是 openrouter / anthropic / openai / deepseek / custom / …

🤖 agent

控制 Agent 循环行为。max_turns=90——单次请求最多调 90 次 LLM。tool_use_enforcement=true——强制 LLM 用工具做事。

🖥️ terminal

命令在哪执行。local（本机）最常用。docker 隔离环境。ssh 远程执行。modal 无服务器。

🗜️ compression

上下文太长时的自动压缩策略。默认：用满 50% 触发，压到 20%。保护最近 20 条消息不被压缩。

🛡️ approvals

manual——危险命令弹确认。smart——LLM 辅助判断风险。off——跳过所有确认（= --yolo）。

🚀 delegation

子代理配置。可单独指定模型（省钱用小模型跑简单任务），控制最大迭代次数。

⚠ 改配置后需要重启：在 Gateway 模式下 /restart，终端模式退出重进。改 config.yaml 不会热加载——因为 System Prompt 需要重建。
📌 本节要点
  • hermes config edit 或 hermes config set key val 修改配置
  • model 段必填——决定 Agent 用哪家 LLM
  • 改配置后需要重启会话（/reset 或重进）
  • 密钥放 .env，不要直接写 config.yaml
4 .env 密钥管理
密钥和配置分离——这是安全底线
# 查看 .env 路径
hermes config env-path

# 典型 .env 内容
OPENROUTER_API_KEY=sk-or-v1-xxxxxxxxxxxxxxxx
ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxxxxxx
DEEPSEEK_API_KEY=sk-xxxxxxxxxxxxxxxx
GEMINI_API_KEY=AIzaxxxxxxxxxxxxxxxx
HERMES_HOME=/opt/data # 可选：自定义数据目录

每个 Provider 有对应的环境变量名。完整列表见下一节 Provider 表格。

🔐 为什么分离？

config.yaml 可以分享、可以入 git（不含密钥）。.env 绝对不能入 git——应该在 .gitignore 中排除。

🔄 热加载

修改 .env 后运行 /reload（终端模式）或在 Gateway 模式 /restart。不需要完全重启。

📌 本节要点
  • 密钥永远放 .env，不要写 config.yaml
  • .env 不入 git——把它加到 .gitignore
  • 每个 Provider 有自己的环境变量名，见 Provider 表格

🔧 Custom Provider：接入任意 OpenAI 兼容端点

如果你想用 Ollama 本地模型、自己部署的 vLLM、或者公司内部的 LLM 网关：

# config.yaml
model:
default: "llama-3.1-70b"
provider: "custom"
base_url: "http://localhost:11434/v1" ← Ollama / vLLM 地址
api_key: "ollama" ← Ollama 不需要真 key，随便填

只要端点兼容 OpenAI 的 /v1/chat/completions 格式，就能用。

📌 本节要点
  • 20+ Provider，API Key / OAuth 两种认证方式
  • OpenRouter 是初学者的最佳起点
  • Custom Provider 可以接任何 OpenAI 兼容端点
  • Provider 可以随时切换，/reset 生效
6 模型选择与切换
挑一个适合你任务和预算的模型

交互式选择

hermes model # 打开交互式模型选择器

命令行切换

# 临时切换（仅本次会话）
hermes chat -m "anthropic/claude-sonnet-4"

# 永久切换（修改 config.yaml）
hermes config set model.default "anthropic/claude-sonnet-4"

# 会话内切换
/model anthropic/claude-sonnet-4 ← 斜杠命令

📊 模型选择指南

🏆 最强综合

anthropic/claude-sonnet-4（通过 OpenRouter）
代码 + 推理 + 工具调用均为顶级。推荐有预算的首选。

💰 高性价比

deepseek/deepseek-chat（DeepSeek V3）
性能接近顶级，价格仅为 1/10。日常开发首选。

🇨🇳 国内友好

deepseek/deepseek-chat / kimi/moonshot-v1 / dashscope/qwen-max
国内网络直连，无需代理。

🔬 推理最强

anthropic/claude-opus-4 / openai/o3
复杂多步推理任务。贵，但出错率最低。

🏠 本地运行

Ollama + custom Provider
完全离线，数据不出本机。需要足够显存（建议 70B+）。

⚡ 快速便宜

google/gemini-2.5-flash / openai/gpt-4o-mini
简单任务、大规模批处理用。速度快、成本极低。

🤔 怎么选？一个实用原则：先用便宜模型跑通流程，确认需求没问题后，切换到强模型做最终执行。这就是"先用 DeepSeek 试，再用 Claude 收尾"的策略。
📌 本节要点
  • hermes model 交互式切换，hermes config set model.default ... 永久切换
  • DeepSeek V3 是性价比之王，Claude Sonnet 4 是能力天花板
  • 可以临时切换模型（-m 参数），不修改 config
  • 子代理可以配不同的模型（省钱策略）
7 凭证池（Credential Pool）
一个 Provider 多个 Key——自动轮转，告别限流

大规模使用时，单个 API Key 可能遇到速率限制（Rate Limit）。Hermes 支持凭证池——给同一个 Provider 配置多个 Key，自动轮转使用。

# 交互式添加凭证
hermes auth add openrouter ← 会提示你输入 API Key

# 查看凭证池
hermes auth list

# 移除某个凭证
hermes auth remove openrouter 0 ← 按索引移除

# 重置耗尽状态（Key 恢复后）
hermes auth reset openrouter

🔁 轮转策略

自动跳过已耗尽的 Key。所有 Key 都耗尽时返回错误。你可以通过 hermes auth reset 手动恢复。

📊 适用场景

① 团队共享 Hermes 实例，每人用各自的 Key
② 多账号规避 Rate Limit
③ 主备切换——一个挂了自动用另一个

8 验证与排障
确认一切正常，出问题了知道看哪里

✅ 验证安装

# 健康检查——一跑就知道哪里有问题
hermes doctor

# 自动修复已知问题
hermes doctor --fix

# 配置检查
hermes config check ← 检查是否有缺失/过期配置项

# 迁移新配置项
hermes config migrate ← 升级后自动补全新配置项

🔍 常见问题速查

现象可能原因解决
hermes: command not found PATH 没配 export PATH="$HOME/.local/bin:$PATH" 加到 ~/.bashrc
LLM 返回 401 / 403 API Key 错误或过期 检查 .env 中对应环境变量，或 hermes auth 重新认证
LLM 返回 429 Rate Limit 等几分钟，或加第二个 Key 到凭证池
工具不出现 Toolset 未启用 hermes tools → 启用对应工具 → /reset
修改配置不生效 没重启会话 Gateway: /restart；终端：退出重进
Gateway 连不上 服务未启动或崩溃 hermes gateway status → hermes gateway restart
💡 万能第一步：遇到任何问题先跑 hermes doctor。它检查：Python 版本、config 完整性、.env 配置、Provider 连通性、工具依赖。大部分问题在这一步就能定位。
9 常见疑问
Q: 我可以同时配多个 Provider 吗？不同任务用不同模型？
可以。config.yaml 里 model.default 是默认模型。你可以临时用 -m 参数指定其他模型，也可以给子代理（delegation）单独配置模型。同一个会话内 /model xxx 切换（需要 /reset）。
Q: 不用 OpenRouter，怎么直接接 Anthropic API？
hermes config set model.provider anthropic，然后在 .env 中设 ANTHROPIC_API_KEY。模型名直接用 claude-sonnet-4-20250514（不带 anthropic/ 前缀）。
Q: config.yaml 和 .env 到底谁优先？
.env 中的环境变量优先级 高于 config.yaml 中的 api_key 字段。所以如果你在 config.yaml 里写了 key，又在 .env 里写了同名环境变量，.env 的值会覆盖。
Q: 怎么知道当前用的是哪个模型？
终端：/model 不带参数显示当前模型名。
或者 hermes config | grep default 查看配置。
Q: 能用 GitHub Copilot 的订阅来跑 Hermes 吗？
可以。hermes model → 选 GitHub Copilot → 按提示完成 OAuth 设备码认证。注意：Copilot 的 token 有速率限制，不适合高频大量调用。
