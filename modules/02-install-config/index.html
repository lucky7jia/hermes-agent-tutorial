1 安装 Hermes
一条命令，60 秒搞定

🐧 Linux / macOS / WSL2

# 一行安装
curl -fsSL https://hermes-agent.nousresearch.com/install.sh | bash

🪟 Windows (PowerShell)

# PowerShell
iex (irm https://hermes-agent.nousresearch.com/install.ps1)

📦 Hermes Desktop

图形界面用户可从官网下载安装包，自带 CLI + 桌面 GUI。

安装脚本做了什么？
① 下载最新版 → ② 解压到 ~/.hermes/hermes-agent/ → ③ 创建 ~/.local/bin/hermes 软链接 → ④ 加入 PATH
不污染系统 Python，不碰全局环境。

📁 安装后

~/.hermes/ 下的完整结构在模块 1 已详述。安装完成时只有源码目录——其余在首次运行后自动创建。

🔧 更新

hermes update 一键更新。重跑安装脚本也能覆盖旧版本，保留配置和会话。

🗑️ 卸载

hermes uninstall 干净卸载。完全清除：卸载后手动删除 ~/.hermes/。

📌 本节要点
  • 一条命令安装，不污染系统 Python
  • ~/.hermes/ 是数据根，~/.local/bin/hermes 是入口
  • hermes update 升级，hermes uninstall 卸载
2 首次运行：hermes setup
装完还不能直接用——需要选模型、配密钥。Setup 向导一步步搞定。
# 交互式配置向导
hermes setup

# 跳到指定段
hermes setup model       # 只配模型
hermes setup terminal    # 只配终端后端
hermes setup gateway     # 只配平台接入
步骤问什么你做什么
1选 Provider从列表选——OpenRouter / Anthropic / OpenAI / DeepSeek / …
2输入 API Key粘贴 Key，自动写入 .env
3选模型从 Provider 的模型列表中选默认模型
4终端后端local（本机）/ docker / ssh——默认 local
5平台接入（可选）配飞书/Telegram/Discord 等，跳过即纯终端模式
🚀 最快上手：有 OpenRouter 账号的话，hermes setup --portal——一套 OAuth 搞定模型+搜索+生图+浏览器，零配置。
3 config.yaml 深度解析
一行 YAML 一个行为——这里是你控制 Hermes 的总开关
hermes config           # 查看当前配置
hermes config edit      # 用编辑器打开
hermes config set section.key value  # 命令行改单项
model:                       ← 模型配置（必填）
  default: "deepseek/deepseek-chat"  ← 默认模型
  provider: "openrouter"         ← Provider
  context_length: 1000000       ← 上下文长度

agent:                       ← Agent 行为
  max_turns: 90            ← 单次最大 LLM 调用次数

terminal:                    ← 终端后端
  backend: "local"          ← local / docker / ssh

compression:                 ← 上下文压缩
  threshold: 0.50          ← 使用率超 50% 触发
  target_ratio: 0.20       ← 压到 20%

memory:                       ← 持久记忆
  memory_enabled: true
  user_profile_enabled: true

approvals:                   ← 命令审批
  mode: "manual"          ← manual / smart / off
⚠ 改配置后需要重启：Gateway /restart，终端退出重进。改 config.yaml 不会热加载——System Prompt 需要重建。
4 .env 密钥管理
# 典型 .env
OPENROUTER_API_KEY=YOUR_API_KEY
ANTHROPIC_API_KEY=YOUR_API_KEY
DEEPSEEK_API_KEY=YOUR_DEEPSEEK_KEY
HERMES_HOME=/home/user/.hermes        # 可选：自定义数据目录

🔐 为什么分离？

config.yaml 可分享可入 git。 .env 绝对不能入 git——加到 .gitignore。密钥和配置分离是安全底线。

⚡ 优先级

.env 环境变量优先级 高于 config.yaml 中的 api_key。同名时 .env 覆盖。

7 凭证池（Credential Pool）
一个 Provider 多个 Key——自动轮转，告别限流
hermes auth add openrouter   # 添加 Key 到凭证池
hermes auth list             # 查看所有凭证
hermes auth reset openrouter  # 恢复耗尽状态的 Key
flowchart TD A["Key 0 · 活跃
***..."] --> D["🔄 轮转选择器
返回活跃的 Key"] B["Key 1 · 备用
***..."] --> D C["Key 2 · 已耗尽
rate limited"] -.->|"跳过"| D D --> E["✅ 始终使用可用 Key"]

🔁 轮转策略

自动跳过已耗尽 Key。全耗尽时返回错误。hermes auth reset 手动恢复。

📊 适用场景

① 团队共享实例各用各 Key
② 多账号规避 Rate Limit
③ 主备切换

9 常见疑问
Q: 能同时配多个 Provider 吗？
可以。model.default 是默认，-m 参数临时切换，子代理可单独配模型。
Q: config.yaml 和 .env 谁优先？
.env 环境变量优先级高于 config.yaml 中的 api_key。
Q: 怎么不用 OpenRouter 直连 Anthropic？
hermes config set model.provider anthropic，.env 设 ANTHROPIC_API_KEY。模型名不带 anthropic/ 前缀。
Q: 能用 GitHub Copilot 订阅跑 Hermes？
可以。hermes model → 选 GitHub Copilot → OAuth 认证。注意有速率限制，不适合高频大量调用。
