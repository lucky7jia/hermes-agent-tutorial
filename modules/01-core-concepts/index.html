🔍 传统 AI（ChatGPT 等）

像一个知识渊博的顾问——你问它答，对话结束。不会主动做事，不记上次说了什么，无法操作你的系统。

🤖 AI Agent（Hermes 等）

像一个能干的员工——你交代任务，它自己规划、执行、检查结果。能调工具、跨会话记忆、定时运行。

Agent 的四个核心能力

1️⃣ 感知

读取输入——消息、文件、API 返回、终端输出。

2️⃣ 推理

理解意图、拆解任务、决定下一步。LLM 的核心价值。

3️⃣ 行动

写文件、跑命令、调 API。不只"想"，还要"做"。

4️⃣ 记忆

把重要信息存下来，下次还能用。没有记忆 = 每次都是第一次见面。

🤔 ChatGPT 有这四种吗？有感知和推理，但没有行动能力和持久记忆。这就是 Agent 和 Chatbot 的本质区别。

1 Hermes Agent 的核心定位
✅ Hermes 是：有记忆、会学习、能独立干活的自主 Agent 进程——一个"数字员工"。

🔓 Provider 无关

20+ Provider，任务中途可切换。

🧠 自我进化

完成复杂任务后自动保存为 Skill。

终端/飞书/Telegram/Discord 等 10+ 平台。

⏰ 自主运行

Cron 定时任务，无人值守。

2 核心引擎：Agent Loop
理解了这个循环，就理解了 Hermes 怎么"想"和"做"

🎬 具体例子

👤

用户说：

"帮我在 ~/projects/myapp 创建 FastAPI 项目，带 /health 端点。"

步LLM 返回Agent 做什么结果
1Tool Call: mkdir建目录成功
2Tool Call: write_file(main.py)写 FastAPI 代码已写入
3Tool Call: write_file(requirements.txt)写依赖已写入
4Tool Call: pip install装依赖成功
5Tool Call: curl /health验证端点{"status":"ok"}
6文本回复→ "项目已创建，/health 正常"
🤔 Agent 不是一次性搞定。它一步一步来——每步都是 LLM 看了上一步结果后决定下一步。感知→推理→行动→观察→再推理→直到完成。

Turn

1 次 LLM 调用 = 1 Turn。Tool Call 是 Turn 内子步骤。

并行 Tool Call

LLM 可一次返回多个，并行执行。

Prompt Caching

System Prompt 不变→可缓存。改配置需 /reset。

Compression

太长时自动压缩。默认 50% 触发→压到 20%。

🔴 Tool Call

LLM 返回 tool_calls → Agent 执行 → 结果追加 → 再次调 LLM（虚线回路）。循环直到 LLM 认为可以回答。上限 max_turns=90。

🟢 Text

LLM 返回纯文本 → 格式化 → 平台适配器投递 → 用户收到。会话结束。等待下一条消息。

7 常见疑问
Q: Hermes vs LangChain？
LangChain 是开发框架，Hermes 是成品 Agent。
Q: 会不会死循环？
max_turns=90 + Compression + LLM 自身判断力三重保护。
Q: 隐私安全？
数据全本地，不上传。可配置脱敏和定期清理。
Q: 能用本地模型？
可以。任何 OpenAI 兼容端点。建议 70B+。
