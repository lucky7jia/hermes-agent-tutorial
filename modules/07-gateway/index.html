模块 10 / 10

Gateway 多平台接入

同一套 Agent 逻辑跑在飞书、Telegram、Discord 等 10+ 平台

预计阅读时间：8 分钟

1 Gateway 是什么？

Gateway 是 Hermes 的多平台消息适配层。它让 Agent 可以同时在飞书、Telegram、Discord、Slack 等多个平台上收发消息——Agent 核心逻辑不变，只换适配器。

flowchart LR A[Agent Loop] --- B[Gateway] B --- C[飞书] B --- D[Telegram] B --- E[Discord] B --- F[Slack] B --- G[微信] B --- H[API Server]
2 支持的平台
平台配置复杂度说明
Telegram低BotFather 创建 Bot → 填入 Token
Discord低创建 Application → Bot Token
Slack中创建 App → OAuth Token
WhatsApp v0.17中Business API 接入
Signal v0.17中Signal CLI 桥接
飞书 (Feishu/Lark)中创建应用 → 获取凭证 → 配置 Webhook
钉钉 (DingTalk) v0.17中企业内部应用接入
企业微信 (WeCom) v0.17中企业微信应用
微信 (WeChat)高需企业微信或第三方桥接
Matrix v0.17中联邦式即时通信协议
Mattermost v0.17中开源企业 IM
Email (SMTP/IMAP)中收发邮件
SMS v0.17中短信收发
API Server低开箱即用的 HTTP API
Webhooks v0.17低事件驱动触发 Agent
3 配置命令
hermes gateway setup    # 交互式配置平台
hermes gateway run      # 前台运行 Gateway
hermes gateway install   # 安装为后台服务
hermes gateway start     # 启动后台服务
hermes gateway stop      # 停止
hermes gateway restart   # 重启（改配置后）
hermes gateway status    # 查看状态
4 命令审批

在 Gateway 模式下（非终端），危险命令需要远程审批。模式：

manual

默认。危险命令弹出确认按钮，用户点击 Approve/Deny。

smart

用辅助 LLM 评估命令风险，低风险自动放行。

off

跳过所有审批 = --yolo。不推荐。
